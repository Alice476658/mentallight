# 心绪微光 · 部署到线上（静态托管）

本项目是纯静态站：`index.html` + `css/` + `js/` + `assets/`，无需编译。部署后你会得到一个固定网址，别人用浏览器打开即可，比本机 `localhost` 少一层「路径/缓存不一致」的困扰。

## 重要说明

1. **Three.js 从 CDN 加载**：首次打开需要能访问 `cdnjs` / `unpkg`（公司内网若屏蔽外网，页面会提示加载失败）。若要完全离线，需要把 Three.js 文件下载到 `vendor/` 并改 `index.html` 里的 `<script src>`。
2. **情绪分析是前端模拟**：关键词 + 打分，不是大模型；输入里要出现对应词（如「开心」「焦虑」）才容易判中。
3. **首页与倾诉是两种视图**：必须从「进入倾诉」或右侧「倾诉」进入完整倾诉场景，背景与粒子才会跟心情一致（代码里用 `app-view-home` / `app-view-pour` 区分）。

---

## 方式 A：Vercel（推荐，简单）

1. 把本文件夹放进 Git 仓库（GitHub / GitLab / Bitbucket 均可）。
2. 打开 [Vercel](https://vercel.com)，用 Git 导入该仓库。
3. **Root Directory** 选项目根（含 `index.html` 的那一层），Framework Preset 选 **Other**，直接 Deploy。
4. 部署完成后用分配的 `*.vercel.app` 域名访问。

仓库根目录已含 `vercel.json`，会对首页设置 `no-store`，减轻「更新了但浏览器还用旧 HTML」的问题。

---

## 方式 B：Netlify

1. 打开 [Netlify](https://www.netlify.com) → Add new site → Deploy manually，把**整个项目文件夹**拖进去；或连接 Git 仓库，`Publish directory` 填 `.`（根目录）。
2. 根目录已含 `netlify.toml`，同样对 `index.html` 做了短缓存策略。

---

## 方式 C：GitHub Pages

1. 在 GitHub 新建仓库，把本文件夹内容 push 上去。
2. 仓库 **Settings → Pages**：Source 选分支（如 `main`），文件夹选 `/ (root)`，保存。
3. 几分钟后用 `https://<用户名>.github.io/<仓库名>/` 访问（若仓库名不是 `用户名.github.io`，路径会多一级仓库名）。

---

## 本地自检（部署前）

- 用 `启动本地预览.bat` 或 `python -m http.server 8080`，不要用「直接双击打开 `index.html` 的 file://」，否则部分浏览器对模块/路径更挑剔。
- 部署后若仍像旧版：用无痕窗口或强制刷新（Ctrl+F5）。
